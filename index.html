The Twelve-Factor App

  • Methodology for building modern, scalable, and maintainable software-as-a-service apps.
  • I. Codebase: One codebase tracked in revision control, many deploys.
  • II. Dependencies: Explicitly declare and isolate dependencies.
  • III. Config: Store config in the environment.
  • IV. Backing Services: Treat backing services as attached resources.
  • V. Build, release, run: Strictly separate build and run stages.
  • VI. Processes: Execute the app as one or more stateless processes.
  • VII. Port binding: Export services via port binding.
  • VIII. Concurrency: Scale out via the process model.
  • IX. Disposability: Maximize robustness with fast startup and graceful shutdown.
  • X. Dev/prod parity: Keep development, staging, and production as similar as possible.
  • XI. Logs: Treat logs as event streams.
  • XII. Admin processes: Run admin/management tasks as one-off processes.

Best Practices

  • Infrastructure as Code (IaaC): Manage and provision infrastructure through code.
  • Security:
    • Use VPC to isolate your resources.
    • Use Security Groups and Firewalls to control traffic.
    • Implement IAM for granular access control.
    • Encrypt data at rest and in transit.
    • Prefer private S3 buckets over public ones.
  • Reproducible Environments: Ensure consistency across development, staging, and production.

High Performance

  • Compute: Choose the right CPU and Memory for your workloads.
  • Networking: Optimize network performance with appropriate configurations.
  • Storage:
    • Select the right storage type (Disk, SSD).
    • Provision IOPS based on application needs.

Resilience

  • Rolling Deployments: Update applications with zero downtime.
  • Reproducible Environments: Quickly recover from failures.
  • GitOps: Use Git as the single source of truth for declarative infrastructure and applications.
  • Disaster Recovery: Plan for and test disaster recovery scenarios.
  • Health Checks: Automatically detect and replace unhealthy instances.

Efficiency & Cost Optimization

  • Planning and Calculation: Estimate costs and plan capacity.
  • Spot Instances: Reduce compute costs for fault-tolerant workloads.
  • S3 Glacier: Archive data for long-term storage at a lower cost.
  • Graviton Processors: Get better price-performance for your workloads.
  • Serverless: Pay only for what you use.
  • Hosted Services: Leverage managed services to reduce operational overhead.

IAM: Identity and Access Management

  • User Management: Manage users and their access to AWS resources.
  • Machine Management: Manage access for applications and services.
  • User Groups: Organize users and apply policies to groups.
  • Roles & Policies: Define permissions and assign them to users and services.

Compliance Strategies

  • Auditing: Track and log all actions taken in your AWS account.
  • Use services like AWS Config and CloudTrail to monitor compliance.

Storage: S3 Deep Dive

  • Why S3?: Scalable, durable, and highly available object storage.
  • Buckets:
    • Private vs. Public: Secure your data by default.
    • Versioning: Keep a history of your objects.
  • Lifecycle Policies: Automatically transition objects to different storage classes.
  • Glacier: Long-term archival storage.

Storage: EBS

  • Elastic Block Store (EBS): Persistent block storage for EC2 instances.
  • Different volume types for different performance needs.

CI/CD on AWS

  • Focus on Continuous Integration (CI) and GitOps.
  • CI: Automate the build and testing of your code.
  • GitOps: Manage your infrastructure and applications through Git.
  • CodeBuild: Fully managed build service.
  • CodePipeline: Continuous delivery service.
  • CodeDeploy: Automate code deployments.

Containers & Kubernetes

  • ECR (Elastic Container Registry): Store, manage, and deploy container images.
  • ECS (Elastic Container Service): Run Docker containers on AWS.
  • EKS (Elastic Kubernetes Service): Run Kubernetes on AWS.
  • Fargate: Serverless compute for containers.

API Management

  • API Gateway: Create, publish, maintain, monitor, and secure APIs at any scale.
  • AppSync: Build scalable applications, including those requiring real-time updates, on a range of data sources.

Observability

  • CloudWatch: Monitor your AWS resources and applications.
  • Logging: Collect and analyze logs from your applications and services.
  • Alerting: Get notified when something goes wrong.

Infrastructure as Code (IaaC)

  • Terraform: Open-source tool for building, changing, and versioning infrastructure safely and efficiently.
  • CloudFormation: AWS service for writing or modeling your infrastructure in templates.
