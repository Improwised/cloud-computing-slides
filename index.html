Day 1: Foundations

The Twelve-Factor App

A methodology for building modern, scalable, and maintainable software-as-a-service apps.
  • I. Codebase: One codebase, many deploys
  • II. Dependencies: Explicitly declare and isolate
  • III. Config: Store in the environment
  • IV. Backing Services: Treat as attached resources
  • V. Build, release, run: Strict separation
  • VI. Processes: Stateless
  • VII. Port binding: Export services via port binding
  • VIII. Concurrency: Scale out via the process model
  • IX. Disposability: Fast startup and graceful shutdown
  • X. Dev/prod parity: Keep environments similar
  • XI. Logs: Treat as event streams
  • XII. Admin processes: Run as one-off tasks
We'll start with the 12-Factor App principles. These are crucial for cloud-native applications. They guide you to create applications that are easy to deploy, scale, and maintain.

    Security: A Deep Dive

    Hussain will lead this section. Security is job zero in the cloud. We'll cover how to control access, secure your network, and ensure compliance. A demo will follow.

    What is cloud security?

    • Protecting data, applications, and infrastructure in the cloud.
    • Ensuring confidentiality, integrity, and availability of resources.
    • Implementing security best practices and compliance standards.
    We'll discuss how AWS helps you secure your resources and what you need to do to keep your applications safe.

    What are some cloud security challenges?

    • Misconfigured access controls and permissions.
    • Data breaches due to insecure APIs or endpoints.
    • Shared responsibility confusion between provider and customer.
    • Insufficient monitoring and logging.
    • Compliance with industry regulations.
    • Managing secrets and credentials securely.
    Cloud security comes with unique challenges. We'll discuss common pitfalls and how to address them using AWS best practices.

    What types of cloud security solutions are available on AWS?

    • IAM (Identity & Access Management): Manage users, roles, and permissions.
    • VPC Security: Network isolation, security groups, and NACLs.
    • Encryption: AWS KMS for key management, S3/EBS encryption, TLS for data in transit.
    • Monitoring & Auditing: CloudTrail, Config, GuardDuty, Security Hub.
    • Web Application Firewall (WAF): Protects against common web exploits.
    • Secrets Management: AWS Secrets Manager and Parameter Store.
    • Compliance Tools: Artifact, Audit Manager, and Inspector.
    AWS offers a wide range of security solutions, from identity management to encryption and compliance. We'll discuss how these tools help you secure your cloud workloads.

    IAM (Identity and Access Management)

    • Manage users, groups, roles, and policies.
    • Principle of Least Privilege: Grant only the necessary permissions.
    IAM is the cornerstone of AWS security. It allows you to control who can access your resources and what they can do with them. We'll cover best practices for setting up IAM policies and roles. Demo for creating a user, roles, policies and assigning permissions.

    Pets vs. Cattle

    Pets vs. Cattle

    • Pets: Servers that are unique, lovingly tended, and hard to replace. You get upset when they die.
    • Cattle: Servers that are identical, created from a template, and designed to be replaced. You don't care if one dies; you just get another one.
    • The cloud model encourages treating servers as cattle, not pets. This is key for scalability and resilience.
    This is a crucial mindset shift when moving to the cloud. We'll discuss why you should aim for immutable, disposable infrastructure.

    Storage Solutions: S3 & EBS

    Hussain will lead this section. We'll cover AWS's main storage offerings‚ÄîS3 and EBS‚Äîexplaining their use cases, features, and best practices. A demo will follow.

    S3 (Simple Storage Service)

    • Object storage for files, backups, and static assets.
    • Highly scalable, durable, and available.
    • Supports lifecycle policies, versioning, and cross-region replication.
    • Integrates with Glacier for archival storage.

    EBS (Elastic Block Store)

    • Block storage for EC2 instances (like a hard drive).
    • Persistent, encrypted, and snapshot-capable.
    • Multiple volume types for performance and cost optimization.
    • Can be resized and attached/detached from instances.
    S3 is ideal for storing files and backups, while EBS is used for persistent storage attached to EC2. We'll discuss when to use each and show a demo of S3 features.

    Common Use Cases

    • S3: Static website hosting, backup and restore, big data analytics, media storage.
    • EBS: Databases, file systems, application data requiring low latency and high throughput.
    We'll highlight typical scenarios for S3 and EBS, helping you choose the right storage for your workload.

    Best Practices

    • Enable encryption for sensitive data (S3 and EBS).
    • Use lifecycle policies to manage costs in S3.
    • Regularly back up EBS volumes using snapshots.
    • Monitor usage and set alerts for unusual activity.
    Security and cost management are key. We'll cover how to protect your data and optimize storage expenses.

    Compute Solutions: EC2 & Lightsail

    Hussain will lead this section. We'll cover AWS's main compute offerings‚ÄîEC2 and Lightsail‚Äîexplaining their use cases, features, and best practices. A demo will follow.

    EC2 (Elastic Compute Cloud)

    • Scalable virtual servers for any workload.
    • Wide choice of instance types (CPU, memory, storage optimized).
    • Supports auto-scaling, load balancing, and custom AMIs.
    • Flexible networking and security options.

    Lightsail

    • Easy-to-use VPS for quick deployments.
    • Pre-configured blueprints for common stacks (WordPress, LAMP, etc.).
    • Simple pricing and management interface.
    • Great for small apps, dev/test, and learning AWS basics.
    EC2 is ideal for flexible, scalable workloads, while Lightsail is perfect for quick, simple deployments. We'll discuss when to use each and show a demo of launching instances.

    Common Use Cases

    • EC2: Web servers, application servers, batch processing, custom environments, enterprise workloads.
    • Lightsail: Small websites, blogs, dev/test environments, quick prototypes, learning AWS basics.
    We'll highlight typical scenarios for EC2 and Lightsail, helping you choose the right compute service for your workload.

    Best Practices

    • Use auto-scaling and load balancing for high availability (EC2).
    • Choose the right instance type for performance and cost optimization.
    • Regularly update and patch your instances.
    • Enable monitoring and set alerts for resource usage.
    • Secure access using IAM roles and security groups.
    We'll cover how to optimize, secure, and manage your compute resources efficiently.

    Monitoring with CloudWatch

    Hussain will lead this section. Monitoring is essential for maintaining the health and performance of your cloud resources. We'll explore CloudWatch features and best practices, followed by a demo.

    CloudWatch Features

    CloudWatch provides both metrics and logs, allowing you to monitor and respond to events in real time. We'll show how to set up alarms and dashboards for proactive monitoring.

    Metrics

    • Time-series data points for system/application variables
    • Examples: CPU usage, network traffic, error rates
    • Built-in metrics for 70+ AWS services
    • Custom metrics for app-specific monitoring

    Logs

    • Centralized log storage from AWS resources & apps
    • View, search, filter, and archive logs
    • Extract metrics from logs using filters
    • Analyze logs interactively with Logs Insights

    Alarms

    • Set thresholds for metrics
    • Receive notifications when thresholds are breached
    • Trigger actions (SNS, EC2, Auto Scaling)

    Dashboards

    • Customizable visualizations of metrics & logs
    • Use graphs, charts, tables to track performance
    • Identify trends and troubleshoot issues
    • Automatic dashboards for AWS services

    Common Use Cases

    • Monitor EC2 instance health and resource usage.
    • Track application errors and performance via logs.
    • Set up alarms for cost management and security events.
    • Visualize service health with custom dashboards.
    We'll highlight typical scenarios where CloudWatch helps you maintain reliability, security, and cost control.

    Best Practices

    • Enable detailed monitoring for critical resources.
    • Set up alarms for key metrics and thresholds.
    • Centralize logs for easier troubleshooting and compliance.
    • Regularly review dashboards and update them as your environment evolves.
    Effective monitoring is proactive, not reactive. We'll discuss how to use CloudWatch to stay ahead of issues and optimize your AWS workloads.

    Containers on AWS

    • ECR: Store and manage your container images.

    Serverless with Lambda and Fargate

    • AWS Lambda: Run code without provisioning or managing servers.
    • AWS Fargate: Run containers without managing servers or clusters.
    • Pay only for the compute time you consume.
    • Ideal for event-driven architectures and microservices.
    • Supports multiple programming languages and runtimes.
    The next evolution of compute. We'll talk about the benefits of serverless and how Lambda works.

    Hussain will explain how to automate everything from code commit to deployment, which is essential for moving fast and staying reliable.

    • Manage and provision infrastructure through code instead of manual processes.

    • CloudFormation: AWS-native IaC service.

    Hussain will demonstrate how to define your infrastructure in code, which is the foundation of reproducible environments and GitOps.
