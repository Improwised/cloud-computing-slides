Day 1: Foundations

The Twelve-Factor App

A methodology for building modern, scalable, and maintainable software-as-a-service apps.

  • I. Codebase: One codebase, many deploys
  • II. Dependencies: Explicitly declare and isolate
  • III. Config: Store in the environment
  • IV. Backing Services: Treat as attached resources
  • V. Build, release, run: Strict separation
  • VI. Processes: Stateless
  • VII. Port binding: Export services via port binding
  • VIII. Concurrency: Scale out via the process model
  • IX. Disposability: Fast startup and graceful shutdown
  • X. Dev/prod parity: Keep environments similar
  • XI. Logs: Treat as event streams
  • XII. Admin processes: Run as one-off tasks
We'll start with the 12-Factor App principles. These are crucial for cloud-native applications. They guide you to create applications that are easy to deploy, scale, and maintain.

Version Control with Git

  • The foundation for collaboration and automation.
  • Tracks changes, allows for branching, and enables CI/CD.
  • GitFlow and other branching strategies are key for team collaboration.
  • Essential for Infrastructure as Code and GitOps.
Everything in modern software development starts with version control. We'll briefly touch on why Git is so important, especially in the context of cloud infrastructure and automation.

Core Cloud Concepts

  • High Performance: Choosing the right CPU, Memory, and Storage (SSD, IOPS).
  • Resilience: Using rolling deployments, health checks, and planning for disaster recovery.
  • Efficiency & Cost Optimization: Leveraging Spot Instances, Graviton processors, and serverless to reduce costs.
These are the pillars of a well-architected cloud environment. We're not just building things, we're building them to be fast, reliable, and cost-effective.

Security: A Deep Dive

IAM: Identity and Access Management

  • Manage users, groups, roles, and policies.
  • Principle of Least Privilege: Grant only the necessary permissions.
  • VPC & Security Groups: Isolate resources and control traffic.
  • Encryption: Protect data at rest and in transit.
  • Compliance: Use tools like AWS Config and CloudTrail for auditing.
Hussain will lead this section. Security is job zero in the cloud. We'll cover how to control access, secure your network, and ensure compliance. A demo will follow.

  • Lifecycle policies for cost management.
  • Glacier for long-term archival.

EBS (Elastic Block Store)

  • Persistent block storage for EC2 instances.
  • Different volume types for various performance needs.
Another section led by Hussain. We'll discuss the two main types of storage in AWS and when to use each. A demo will showcase S3 features.

Monitoring with CloudWatch

  • Collect metrics, logs, and traces.
  • Create alarms to be notified of issues.
  • Build dashboards to visualize the health of your applications.
  • Demo: Setting up a CloudWatch alarm.
If you can't measure it, you can't improve it. Hussain will show how to use CloudWatch to keep an eye on your AWS resources.

Pets vs. Cattle

  • Pets: Servers that are unique, lovingly tended, and hard to replace. You get upset when they die.
  • Cattle: Servers that are identical, created from a template, and designed to be replaced. You don't care if one dies; you just get another one.
  • The cloud model encourages treating servers as cattle, not pets. This is key for scalability and resilience.
This is a crucial mindset shift when moving to the cloud. We'll discuss why you should aim for immutable, disposable infrastructure.

Containers on AWS

  • ECR: Store and manage your container images.
  • ECS: A fully managed container orchestration service from AWS.
  • EKS: Managed Kubernetes service.
  • Fargate: Serverless compute for containers. No need to manage servers.
  • Demo: Pushing an image to a public ECR repo.
Hussain will take over to discuss containers, a cornerstone of modern application deployment. We'll cover the different ways to run containers on AWS and demo ECR.

Serverless with AWS Lambda

  • Run code without provisioning or managing servers.
  • Pay only for the compute time you consume.
  • Event-driven architecture.
  • Ideal for microservices, data processing, and more.
The next evolution of compute. We'll talk about the benefits of serverless and how Lambda works.

CI/CD & GitOps

  • CI/CD (Continuous Integration/Continuous Deployment): Automate your software delivery pipeline.
  • GitOps: Use Git as the single source of truth for declarative infrastructure and applications.
  • AWS Tools: CodeCommit, CodeBuild, CodeDeploy, CodePipeline.
  • Demo: A look at a simple CI/CD pipeline.
Hussain will explain how to automate everything from code commit to deployment, which is essential for moving fast and staying reliable.

Infrastructure as Code (IaC)

  • Manage and provision infrastructure through code instead of manual processes.
  • Terraform: Open-source, multi-cloud tool.
  • CloudFormation: AWS-native IaaC service.
  • Enables versioning, reusability, and automation of your infrastructure.
  • Demo: A simple Terraform configuration.
Hussain will demonstrate how to define your infrastructure in code, which is the foundation of reproducible environments and GitOps.
